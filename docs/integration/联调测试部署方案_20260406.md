本轮联调继续采用下面的部署拆分：

- 云上阿里云服务器：前端 + 后端 + SRS / HTTP-FLV

## 3. 给前后端同学的联调口径

前端不需要直连模型服务。

前端只需要：

1. 调后端开始检测接口
2. 播放云上视频流
3. 订阅后端 WebSocket 结果
4. 在页面上叠加检测框

后端负责：

1. 调用模型端 `/v1/inference/stream/start`
2. 把 `sourceUrl`、`sessionId`、`callbackToken` 下发给模型端
3. 接收模型端回调 `/api/v1/detect/model/result`
4. 再把结果推给前端

## 4. 推荐本轮联调参数

由于当前机器上 `torch.cuda.is_available() = False`，本轮按 CPU 联调口径执行，先保证链路稳定：

- `sourceUrl`: `http://8.137.107.232:8088/live/001.flv`
- `sample_fps`: `1`
- `pair_layout`: `side_by_side_h`
- `rgb_position`: `left`
