# 模型端稳定部署方案（2026-04-07）

## 1. 目标

当前 `cloudflared quick tunnel` 只适合临时联调，不适合反复复用。稳定方案要解决三件事：

- 模型端地址固定
- 服务进程自动拉起和自动重启
- 不再依赖“有人挂着终端会话”

## 2. 推荐方案

推荐采用：

`本机模型服务 + 阿里云服务器 frp 固定出口 + systemd 守护`

原因：

- 你们已经有云服务器 `47.108.223.231`
- 不要求额外购买域名
- 地址可以固定成 `http://47.108.223.231:18000`
- 后端环境变量只需要改一次
- 稳定性和可控性都比 `quick tunnel` 高

## 3. 建议架构

链路如下：

1. 本机 `uvicorn` 监听 `127.0.0.1:18000`
2. 本机 `frpc` 把 `127.0.0.1:18000` 映射到云服务器固定端口
3. 云服务器 `frps` 对外暴露固定入口，例如 `18000`
4. 后端固定调用：
   - `http://47.108.223.231:18000/v1/inference/stream/start`
   - `http://47.108.223.231:18000/v1/inference/video/start`
   - `http://47.108.223.231:18000/v1/inference/session/stop`
   - `http://47.108.223.231:18000/v1/inference/session/{session_id}`

## 4. 本机侧落地

### 4.1 本机一键渲染

先在本机执行：

```bash
FRP_SERVER_ADDR=47.108.223.231 \
FRP_SERVER_PORT=7000 \
REMOTE_MODEL_PORT=18000 \
bash /home/liujuncheng/rgbt_uav_detection/scripts/install_stable_local.sh
```

默认会生成：

- `~/.config/rgbt_uav_detection/frp_auth_token`
- `~/.config/rgbt_uav_detection/frpc.toml`
- `~/.config/systemd/user/rgbt-model.service`
- `~/.config/systemd/user/rgbt-frpc.service`

### 4.2 本机模型守护

模型端实际由下面脚本启动：

```bash
bash /home/liujuncheng/rgbt_uav_detection/scripts/start_model_stable_local.sh
```

它会固定监听：

- `127.0.0.1:18000`

并使用：

- [deploy_stable.yaml](/home/liujuncheng/rgbt_uav_detection/configs/deploy_stable.yaml)

### 4.3 本机 FRP 客户端守护

生成后的 `frpc.toml` 会把：

- `127.0.0.1:18000`

映射到云服务器：

- `47.108.223.231:18000`

本机 `frpc` 可通过下面脚本下载：

```bash
FRP_VERSION=<指定版本> \
bash /home/liujuncheng/rgbt_uav_detection/scripts/download_frp.sh client
```

## 5. 云服务器侧落地

### 5.1 frps

云服务器通过下面脚本安装 `frps`：

```bash
REMOTE_HOST=8.137.107.232 \
REMOTE_USER=<你的云服务器用户名> \
FRP_AUTH_TOKEN="$(cat ~/.config/rgbt_uav_detection/frp_auth_token)" \
FRP_DASHBOARD_PASSWORD="<自定义密码>" \
bash /home/liujuncheng/rgbt_uav_detection/scripts/install_stable_remote_frps.sh
```

默认会在云服务器落地：

- `/opt/rgbt_uav_detection/bin/frps`
- `/etc/rgbt_uav_detection/frps.toml`
- `/etc/systemd/system/rgbt-frps.service`

### 5.2 安全组

至少放通：

- `7000/tcp` 给 `frpc` 连接
- `18000/tcp` 给前后端联调访问

如果后面要只给后端访问，也可以把 `18000` 收窄到指定来源 IP。

## 6. 后端切换方式

后端把模型基址固定成：

```text
MODEL_BASE_URL=http://47.108.223.231:18000
```

对应接口保持不变，不需要改请求体字段：

- `/v1/health`
- `/v1/model/info`
- `/v1/inference/video/start`
- `/v1/inference/video/stop`
- `/v1/inference/stream/start`
- `/v1/inference/stream/stop`
- `/v1/inference/session/{session_id}`

## 7. 稳定方案切换顺序

建议按下面顺序切：

1. 保持当前 `quick tunnel` 继续联调
2. 本机执行 [install_stable_local.sh](/home/liujuncheng/rgbt_uav_detection/scripts/install_stable_local.sh)
3. 云服务器执行 [install_stable_remote_frps.sh](/home/liujuncheng/rgbt_uav_detection/scripts/install_stable_remote_frps.sh)
4. 本机执行：
   - `systemctl --user daemon-reload`
   - `systemctl --user enable --now rgbt-model.service`
   - `systemctl --user enable --now rgbt-frpc.service`
5. 用 `http://47.108.223.231:18000/v1/health` 验证
6. 后端把 `MODEL_BASE_URL` 改成固定地址
7. 验证 `stream/start -> callback -> websocket` 整条链路
8. 确认无误后，下线 `quick tunnel`

## 8. 备选方案

如果后面你们有自己的域名，优先升级为：

`cloudflared named tunnel + 固定域名 + systemd`

它的优势是：

- 自带 HTTPS
- 域名固定
- 不暴露额外公网端口

但前提是：

- 你们有 Cloudflare 账号
- 你们有可托管到 Cloudflare 的域名

在没有这些前提时，`frp` 是更快落地的稳定方案。
