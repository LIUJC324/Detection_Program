# Changes Archive

`docs/changes/` stores one folder per non-trivial change set.

## Naming Rule

- Folder name: `YYYYMMDD_topic`
- Required file: `README.md`

## What To Put Here

1. 本轮为什么改
2. 改了哪些代码 / 配置 / 文档
3. 当前结果或验证结论
4. 下一步怎么继续

## What Should Not Only Live Here

下面这些仍然要同步回对应主题目录，而不是只写在变更归档里：

1. 训练事实与模型结论：`docs/training/`
2. 联调接口与流程口径：`docs/integration/`
3. 部署与运行说明：`docs/ops/`
4. 架构图与图示说明：`docs/architecture/`
